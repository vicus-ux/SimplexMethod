# Симплекс-метод с правилом Блэнда

Реализация универсального симплекс-метода с правилом Блэнда для решения задач линейного программирования в канонической форме.

### Каноническая задача ЛП
Найти:
min f = c₀ + c₁x₁ + c₂x₂ + ... + cₙxₙ
при условиях:
b₁ = a₁₁x₁ + a₁₂x₂ + ... + a₁ₙxₙ
b₂ = a₂₁x₁ + a₂₂x₂ + ... + a₂ₙxₙ
...
bₘ = aₘ₁x₁ + aₘ₂x₂ + ... + aₘₙxₙ
x₁, x₂, ..., xₙ ≥ 0

В матричной форме:
min f = c₀ + cᵀx
Ax = b
x ≥ 0

### Прямо допустимая таблица
Таблица называется прямо допустимой, если:
- Матрица A содержит столбцы, составляющие единичную подматрицу E ранга m
- cⱼ = 0 для базисных столбцов
- bᵢ ≥ 0 (i = 1, ..., m)

### Правило Блэнда

- Выбрать ведущий столбец с наименьшим номером среди столбцов с отрицательной стоимостью
- Выбрать ведущую строку так, чтобы вывести из базиса переменную с наименьшим номером

Алгоритм:
1. Если таблица двойственно допустима (все cⱼ ≥ 0) → СТОП
2. Найти наименьшее k ≥ 1: cₖ < 0
3. M = {i : aᵢₖ > 0}
4. Если M = ∅ → целевая функция неограничена → СТОП
5. α = min{bᵢ/aᵢₖ : i ∈ M}
6. I = {i ∈ M : bᵢ/aᵢₖ = α}
7. Найти s: J[s] = min{J[i] : i ∈ I}
8. Выполнить преобразование с ведущим элементом aₛₖ, J[s] := k
9. Перейти к шагу 1

Поступает файл .txt вида:
min (или max)
c₁ c₂ ... cₙ
a₁₁ a₁₂ ... a₁ₙ <= b₁ (или >=, =)
a₂₁ a₂₂ ... a₂ₙ <= b₂
...
aₘ₁ aₘ₂ ... aₘₙ <= bₘ
x₁, x₂, ..., xₙ >= 0

Итерационный процесс
На каждой итерации:
Проверка оптимальности - все коэффициенты в целевой строке ≥ 0
Выбор ведущего столбца - первый столбец с отрицательным коэффициентом
Выбор ведущей строки по правилу Блэнда:
Находим допустимые строки (aᵢₖ > 0)
Вычисляем отношения bᵢ/aᵢₖ
Выбираем строку с минимальным отношением
При равенстве - выбираем переменную с наименьшим номером
Преобразование Гаусса-Жордана с ведущим элементом
Обновление базиса

допы:
Метод Гаусса-Жордана
Элементарные преобразования строк:
Перестановка строк с номерами i и k: (i) ↔ (k)
Умножение i-й строки на число t: (i) ← t·(i)
Прибавление к i-й строке произведения строки k и числа t: (i) ← (i) + t·(k)

Начальный базис
Если в матрице A нет единичной подматрицы, используются дополнительные переменные:
Для ограничений типа "≤" добавляем slack-переменные
Для ограничений типа "≥" добавляем surplus-переменные
Для ограничений типа "=" добавляем искусственные переменные

Критерии остановки
Оптимальность: все cⱼ ≥ 0 в целевой строке
Неограниченность: M = ∅ на шаге 4 алгоритма
Отсутствие допустимого решения: невозможно построить начальный базис с b ≥ 0
